## Arco narrativo do Mundo 2

> Vovó: "O cajado não atravessa quilómetros — atravessa milénios. Ao
> longo do Vale do Nilo, três reinos negros mudaram o mundo. Vamos
> visitar pela ordem do tempo."

3 fases temporais em ordem cronológica (não geográfica). Geograficamente
o arco vai do **norte** (Egito) para **sul/leste** (Etiópia) — coincide
com a cronologia cultural. Sem framing "descer rio" (Nilo flui de sul
para norte; confusão evitada).

Mecânica recorrente: **ler hieróglifos / símbolos** — interpretar
pictogramas para abrir caminho, em vez de revelar memórias só pelo
cajado. Reaproveita overlay `#enigma` existente com 3 campos opcionais:
`attempts` (padrão 1), `hints[]` (array progressivo), `nonBlocking:
true` (desbloqueia depois de 3 tentativas).

## Fase 2.1 — Cheias do Kemet

**Período**: ~3000 a.C. (Reino Antigo, Egipto Faraónico).

**Story beat**: Lucinha aterra perto do Nilo. As cheias estão a subir.
Um escriba mostra-lhe que ler símbolos não é arte — é sobrevivência.

### Learning objectives

- Kemet (Egipto Antigo) = "Terra Negra" pela cor do solo fértil pós-cheia.
- Hieróglifos = escrita + pictograma + som. Não é "desenho mágico".
- Cheia anual do Nilo previa colheitas. Calendário e astronomia
  africanos nasceram aqui.
- O Egipto Antigo é parte da história africana, não excepção dela.

### Layout (x: -5 → 50)

- Setor 1 (x: -5 → 12): margem oeste do Nilo. Solo seco, palmeiras.
- Setor 2 (x: 12 → 32): zona de cheia. Plataformas de pedra com
  símbolos hieroglíficos visíveis nos topos.
- Setor 3 (x: 32 → 50): templo, escriba, estátua do faraó (Núbia
  começa logo a sul).

### Mecânica nova — "Cheia rítmica"

- Água do Nilo sobe e desce em ciclo de 8 segundos (4s baixa, 100ms
  rising transition, 4s alta, 100ms falling transition).
- **Sistema delta-time accumulator** (não setTimeout/setInterval) para
  resistir a frame drops mobile (thermal throttling):
  ```js
  // estilo IIFE existente, não classes ES6
  var floodSystem = {
    cycleDuration: 8000, accumulated: 0,
    state: 'low', plats: []
  };
  function updateFlood(dt){
    floodSystem.accumulated += dt * 1000;
    var phase = floodSystem.accumulated % floodSystem.cycleDuration;
    // Histerese 200ms previne flicker no threshold
    var s = phase < 3900 ? 'low'
          : phase < 4000 ? 'rising'
          : phase < 7900 ? 'high'
          : 'falling';
    if(s !== floodSystem.state){ /* notify plats */ }
  }
  ```
- Plataformas baixas (`riverPlats`) submergem quando água sobe — só
  são pisáveis na fase baixa.
- Cair em água submersa = hazard `S.deathCause='water'` → death pause
  + respawn na última plataforma segura (`S.lastSafeX` / `S.lastSafeY`).
- **Warning system (1.5s antes da submersão)**:
  - Estado intermédio `'warning'` aos 2400ms do ciclo (1.5s antes do
    rising às 3900ms).
  - Visual: plataformas `riverPlats` ganham aura azul cintilante +
    pequenas ondas/ripples na borda da água a aproximar.
  - Audio: `cheia-warning.mp3` curto (sino abafado) aos 2400ms.
  - Em mobile: HUD pulsa azul translúcido 1s.
- Audio sync: SFX `cheia.mp3` dispara em 'rising'; reset em 'falling'.

**Mecânica nova — "Ler hieróglifo"** (extensão `#enigma`):
- Algumas portas/passagens têm cartouche com 3 símbolos Unicode
  (`𓂀 𓃻 𓆣` etc — sem assets externos por defeito).
- **Fallback Unicode** (dispositivos antigos / browsers sem font
  Egyptian Hieroglyphs):
  - Detecção via canvas measureText em `init()`: render `𓂀`,
    se largura == fallback box width (`tofu`) → activate PNG fallback.
  - PNG fallback: `assets/world2-1/hieroglyphs/<symbol>.png`
    (8 símbolos pré-renderizados a 64×64 transparente).
  - CSS `@font-face` opcional: incluir Noto Sans Egyptian Hieroglyphs
    (Google Fonts, 90KB woff2) só na fase 2.1.
- Aproximar + **E** abre overlay `#enigma` com 3 campos novos:
  - `attempts: 3` — até 3 tentativas sem custo de vida.
  - `hints: [hint1, hint2]` — revelados após tentativa 1 e 2.
  - `nonBlocking: true` — após 3 falhas, revela resposta correta +
    desbloqueia passagem (não bloqueia progresso).
- Acertar dispara `applyEnigmaReward('passagem')` existente.
- Re-usa visual `.e-meta` / `.e-scene` / `.e-options` — sem CSS novo.

### Hazards

- **Crocodilo do Nilo**: já existe em 1.2; reaproveitar com sprite

- **Cheia submersão**: water hazard ciclo 8s (já novo).

## Fase 2.2 — Pirâmides de Meroé

**Período**: ~700 a.C. – 350 d.C. (Reino de Kush tardio).

**Story beat**: Sul de Tebas, Lucinha atravessa o Sudão. Mais pirâmides
do que no Egipto — só mais pequenas. Aqui reinaram rainhas guerreiras.

### Learning objectives

- Meroé teve **mais de 200 pirâmides** — mais que o Egipto.
- **Kandakes** = título das rainhas-guerreiras de Kush. Comandavam
  exércitos.
- Ferro Meroé = produção industrial antiga (séculos antes da Europa).
- Escrita meroítica = sistema próprio, ainda parcialmente indecifrado.

### Layout (x: -5 → 55, labirinto interno)

- Setor 1 (x: -5 → 15): savana com pirâmides pequenas a Sul.
- Setor 2 (x: 15 → 38): **interior de pirâmide** — corredores
  estreitos, baixa visibilidade (fog densa próxima).
- Setor 3 (x: 38 → 55): câmara do tesouro + saída.

### Mecânica nova — "Tochas que apagam"

Implementação **three.js-native** (sem canvas 2D fog manual):

- `scene.fog = new THREE.FogExp2(0x000000, 0.15)` — escuridão global
  no setor 2.
- Cada tocha = `THREE.PointLight` com `distance=4`, `decay=2`,
  `intensity=1.5`. GPU calcula iluminação automaticamente.
- Tochas em x: 17, 24, 31, 35. Acendem ao passar (proximity < 1.5).
- Lifetime 8s; quando expira, `light.intensity` faz fade 0.5s → 0.
- Reacender com **B**: `light.intensity = 1.5`, reset timer.
- Performance: GPU shader, não custo CPU por frame (vs canvas radial
  gradient).

**Mecânica nova — "Empurrar bloco"**:
- Blocos de pedra movíveis em x: 22, 30. Player encosta + segura
  ARROW → empurra 1 unidade. Bloqueia/desbloqueia passagens.
- **Feedback tátil/visual** (crítico em mobile onde segurar direcional
  é menos intuitivo):
  - Dust particles contínuas debaixo do bloco enquanto move
    (reaproveita `spawnDust(blockX, .1, .4)` a cada 100ms).
  - Tilt ligeiro do sprite player durante empurrar (`luci.material.rotation`
    ±0.08 rad) — comunica esforço.
  - SFX `bloco-arrastar.mp3` loop curto enquanto empurra; cut on
    release.
  - Toast contextual primeira vez: "📦 Segura ← ou → para empurrar."
  - HUD hint mobile: botão direcional pulsa enquanto player encosta
    em bloco empurrável.

### Hazards

- **Sombra do Ferro** 👻 (não múmia — Meroé não tinha tradição
  mumificação egípcia; pirâmides eram abobadadas com enterros
  simples). Lore: espírito guardião dos ferreiros e mineiros que
  trabalharam exaustos nos fornos. Patrol horizontal (reaproveita
  sistema crocodile). Death cause: `'sombra'` ("👻 A sombra do ferro
  alcançou-te").
- **Repelida por luz**: a Sombra **foge de tochas acesas**. Em cada
  frame, calcular distância da sombra à tocha acesa mais próxima:
  ```js
  for each light in active torches:
    if (distance(sombra, light) < light.distance * 0.8):
      sombra.fleeDir = sign(sombra.x - light.x)
      sombra.speed *= 1.5  // recua mais rápido
  ```
  Resultado: jogador usa tochas como **proteção tática**, não só
  visibilidade. Mecânica de tochas (luz) e hazard (sombra) ficam
  acopladas — design coerente, recompensa estratégia.
- **Pedras a cair** 🪨: 3 spots no setor 2; sombra avisa 1s antes,
  depois pedra cai = -1 vida. Death cause: `'pedra'`.

## Fase 2.3 — Estelas de Aksum

**Período**: séc. I – VII d.C. (Reino de Aksum, Etiópia/Eritreia).

**Story beat**: Lucinha sobe às montanhas etíopes. Aksum, reino
comerciante, cristão antes da Europa do Norte ouvir falar de Cristo.

### Learning objectives

- Aksum era um dos **4 grandes impérios** do mundo antigo no século III
  (com Roma, Pérsia, China).
- Aksum cunhou moeda própria — uma das primeiras potências africanas
  com economia monetária.
- Cristianismo etíope = séc. IV (antes da maior parte da Europa).
- Comércio Mar Vermelho ligava África, Arábia, Índia, Roma.

### Layout (x: -5 → 60, vertical climb)

- Setor 1 (x: -5 → 15): porto Adulis, navios mercantes.
- Setor 2 (x: 15 → 40): estela vertical gigante (escalada — uso de
  saltos sucessivos em plataformas erguidas em zigzag).
- Setor 3 (x: 40 → 60): mosteiro montês, escriba Ge'ez, ícone
  cristão antigo.

### Mecânica nova — "Escalar estela" (sistema de âncoras)

Plataformas zigzag vertical (y: 0 → 16). 3 âncoras (checkpoints)
embutidas:

```js
var verticalClimb = {
  anchors: [
    { y: 4,  activated: false },
    { y: 8,  activated: false },
    { y: 12, activated: false }
  ],
  currentAnchor: 0
};

function onReachAnchor(idx){
  verticalClimb.anchors[idx].activated = true;
  // Visual: estela brilha breve, snd('w') discreto
}

function onFallFromClimb(){
  // Reset para última âncora ativada (não chão)
  var resetY = 0;
  for(var i = verticalClimb.anchors.length-1; i >= 0; i--){
    if(verticalClimb.anchors[i].activated){ resetY = verticalClimb.anchors[i].y; break; }
  }
  // Penalidade leve: -1 cauri (não vida; sunk cost mais suave)
  if(S.cc > 0) S.cc--;
  S.y = resetY + 1;
}
```

**Vento variável** (não fixo 3s):
- Padrão: 2s fraco → 1s forte → 2s fraco → 1s pausa.
- Indicador visual 1s antes do vento forte (folhas a mover, som leve).
- Player pode segurar **↓** para "agarrar" plataforma (resiste vento
  forte; perde meio segundo de tempo).

**Mecânica nova — "Moeda Aksum"**:
- Moedas especiais (3 unidades, distintas dos cauris) revelam selos
  reais do rei Ezana. Coletar todas dá `reward: 'ferramenta'`
  (símbolo régio).
- **HUD diferenciado** — crítico para não confundir com cauris:
  - Novo state field `S.aksumCoins` (não usa `S.cc`).
  - Sprite mesh dourado **achatado** (disco, não toroide) com cruz
    copta gravada — visualmente distinto do cauri (concha).
  - Ícone HUD específico em `#hud-right`: `<div class="hi"><div class="ico ico-a">⚪</div><span id="ac">0/3</span></div>`
    com cor de fundo cobre/dourado (`#a8763a`).
  - Counter `0/3` (não simples contagem) reforça que é coleção
    fechada, não acumulação livre.
  - Phase 2.3 only: HUD acrescenta este `hi` ao carregar PHASE; outras
    fases não o veem.
  - SFX coleção diferente do cauri (`moeda-aksum.mp3` mais metálico

## Fallback áudio — modos não-ocidentais

Suno v5 tende a harmonizar em major/minor ocidental. Modos modais
árabes (maqam hijaz, kurd) e etíopes (tizita, anchihoy) ficam
genéricos. Estratégia em 3 níveis:

1. **Suno v5 com prompt modal explícito**:
   - 2.1: `phrygian mode, ancient egyptian harp pentatonic, dorian-influenced melody, NOT major scale`.
   - 2.2: `arabic maqam hijaz, oud microtonal bends, iqa'at rhythm wahda, NOT western scale`.
   - 2.3: `ethiopian tizita mode, krar lyra pentatonic, masinko bowed, NOT major scale`.
   - 3 takes por fase; iteração até soar autêntico.

2. **Se Suno falhar (>3 takes ainda genérico)** — sample libraries:
   - **Splice** (subscription): pacotes "Middle Eastern", "African
     Vibes", "Ethiopian Jazz".
   - **Spitfire Audio LABS** (free): Ethnic strings, percussion.
   - **Orchestral Tools SINE** (free tier): folk instruments.
   - Workflow: layering manual em LMMS / Reaper / Audacity (~6h
     extra por track).

3. **Último recurso** — encomendar a músico humano (etnomusicólogo /
   produtor world music). ~100-200€/track.

Estimativa 14h cobre níveis 1+2 (iteração Suno + layering parcial se
necessário). Nível 3 requer budget adicional.

## Respawn consistente (transversal Mundo 2)

Todas as mortes em Mundo 2 respawnam na **última plataforma segura**
(`S.lastSafeX`, `S.lastSafeY`) — não no início da fase.

- `S.lastSafe{X,Y}` atualizado cada frame em que `S.gnd === true`
  E `S.y > -2` (não em queda) E está em plataforma estável (não em
  `riverPlat` no momento de submersão).
- `loseLife()` consulta `lastSafeX/Y` em vez de `S.x=0; S.y=5`
  hardcoded.
- Backwards compat 1.x: se phase data tiver `respawnMode:'start'`,
  reverte ao comportamento Mundo 1 (respawn origem).
- Death pause sequence existente (sprite tipped + banner) corre antes
  do respawn — sem mudança ao fluxo de UX.

Benefício: progresso preservado, menos frustração em fases longas
(2.2 labirinto, 2.3 escalada).

## Decisões técnicas aplicadas

- **Múmia Meroé → Sombra do Ferro** (autenticidade: Meroé não tinha
  tradição mumificação egípcia; lore conecta aos ferreiros/mineiros).
- **"Descer o Nilo" → "atravessar Vale do Nilo em três tempos"**
  (Nilo flui sul→norte; metáfora descida geograficamente errada).
- **Tochas: three.js PointLight + FogExp2** (não canvas 2D fog).
- **Cheia: delta-time accumulator** (não setInterval — frame drop
  safe).
- **Escalada: âncoras + -1 cauri** (não respawn total; sunk cost
  suave).
- **Vento variável** (não fixo 3s); indicador 1s antes do strong
  gust; segurar ↓ resiste.
- **Death causes novos**: `'water'`, `'sombra'`, `'pedra'`. Labels já
  preparados em `deathLabel()` ou trivial de adicionar.
- **Audio**: Opus em vez de MP3 (50% menor, mesma qualidade,
  suporte universal browsers modernos).
- **Warning cheia 1.5s antes** (aura azul + ripples + sino + HUD
  pulse mobile) — janela suficiente para reacção.
- **Hieróglifo Unicode fallback**: detect via canvas measureText;
  fallback PNG 64×64 pré-renderizados em `assets/world2-1/hieroglyphs/`.
- **Sombra do Ferro repelida por luz** — tochas viram proteção
  tática, mecânicas luz+hazard acopladas.
- **Respawn consistente** — todas as mortes em Mundo 2 usam
  `lastSafeX/Y` (última plataforma segura, não origem). Phases 1.x
  mantêm respawn origem via flag `respawnMode:'start'`.
